# 練習 2：變數宣告與 TYPE / LIKE

## 學習目標

- 會用 `DATA ... TYPE` 搭配內建型別宣告變數：`i`、`string`、`c LENGTH n`、`p LENGTH n DECIMALS m`、`d`
- 理解 `TYPE`（參照**型別**）與 `LIKE`（參照**既有變數**）的差異與使用時機
- 會用 `CONSTANTS` 宣告常數
- 會用 `=` 指定值、做簡單四則運算
- 認識第一個系統欄位 `sy-datum`

## 事前準備

建立程式 `ZR_TR02_<你的姓名縮寫>`，套件 `$TMP`。

## 題目需求

1. 用 `TYPE` 宣告五個變數：上課次數（整數）、課程名稱（字串）、課程代碼（固定 4 碼字元）、定價（數值、小數 2 位）、今天日期（日期）
2. 用 `LIKE` 宣告兩個變數：含稅價（型別跟定價一樣）、課程名稱備份（型別跟課程名稱一樣）
3. 宣告常數稅率 `0.05`（不可修改的值要用 `CONSTANTS`）
4. 在 `START-OF-SELECTION` 中給值：日期用 `sy-datum`，含稅價 = 定價 ×（1 + 稅率）
5. 用 `WRITE` 把所有變數輸出

## 預期輸出（範例，日期依執行當天而定）

```
課程名稱： 基礎 ABAP Training
課程代碼： TR02
上課次數：          3
今天日期： 03.07.2026
定價    ：           1,250.50
含稅價  ：           1,313.03
備份字串： 基礎 ABAP Training
```

## 思考題

1. 如果之後「定價」的型別從小數 2 位改成 3 位，「含稅價」需要跟著改宣告嗎？為什麼？
2. `CONSTANTS` 宣告時不給 `VALUE` 會發生什麼事？（動手試試，看錯誤訊息）

## 提示

- `TYPE` 用在「我知道要什麼型別」；`LIKE` 用在「我要跟某個變數一模一樣」——來源改了會自動跟著改，這就是思考題 1 的答案方向
- 數值字面值帶小數時要用引號：`gv_price = '1250.50'.`

## 答案

見 `zr_tr02_type_like.prog.abap`（SAP 端程式 `ZR_TR02_TYPE_LIKE`）。
