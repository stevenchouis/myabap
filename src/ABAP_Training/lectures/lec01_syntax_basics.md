# 講義 1：ABAP 語法基礎

> 對應練習：[ex01](../ex01_syntax_basics.md)｜答案程式：`ZR_TR01_SYNTAX_BASICS`

## 本講重點

- ABAP 程式怎麼建立、怎麼執行（SE38 / Executable Program）
- statement 以句點 `.` 結束，可自由跨行
- 兩種註解：`*`（整行）與 `"`（行內）
- 冒號 `:` 鏈式寫法
- 關鍵字不分大小寫與 Pretty Printer
- `WRITE` 輸出與 `/` 換行

## 1. ABAP 程式的執行環境

ABAP 程式不是存在你電腦裡的檔案，而是存在 SAP 系統資料庫中的「Repository 物件」，在應用伺服器上執行。本課程用到的工具：

| 交易代碼 | 用途 |
|---|---|
| SE38 | ABAP Editor：建立、修改、執行程式 |
| SE80 | Object Navigator：總覽套件下所有物件 |
| SA38 | 只能「執行」程式（給一般使用者） |

建立程式的固定步驟（本課程所有練習共用）：

1. SE38 → 輸入程式名（客戶自開發一律 `Z` 或 `Y` 開頭，課程用 `ZR_TRnn_姓名縮寫`）→ Create
2. 類型（Type）選 **Executable Program**（可直接 F8 執行的報表程式）
3. 套件（Package）填 `$TMP`（Local Object，不產生傳輸請求，適合練習）
4. 編輯器中撰寫 → `Ctrl+F2` 語法檢查 → `Ctrl+F3` 啟用（Activate）→ `F8` 執行

> 觀念：**存檔 ≠ 啟用**。程式有「作用中（active）」與「修訂中（inactive）」兩個版本，執行的是啟用過的版本。改完沒啟用，跑起來還是舊的——初學最容易困惑的地方。

每支 Executable Program 的第一個 statement 一定是：

```abap
REPORT zr_tr01_syntax_basics.
```

## 2. Statement 與句點

ABAP 的最小單位是 statement（陳述式），規則只有一條：**以句點 `.` 結束**。換行、縮排、多個空白都不影響意義，一個 statement 可以拆成很多行寫：

```abap
WRITE 'Hello ABAP!'.

* 跟上面那行完全等價：
WRITE
  'Hello ABAP!'
  .
```

因此 ABAP 沒有「一行一句」的概念——**編譯器只認句點**。這帶來兩個常見錯誤：

- 忘記句點：錯誤訊息往往指到「下一行」，因為編譯器把下一行當成同一句的延續。看到怪錯誤，先檢查上一行結尾。
- 句點打成逗號：鏈式寫法（見第 4 節）中間才用逗號，最後一定是句點。

## 3. 註解

| 寫法 | 規則 | 範例 |
|---|---|---|
| `*` 整行註解 | 星號必須在**第 1 個字元**（行首），整行都是註解 | `* 這是整行註解` |
| `"` 行內註解 | 從雙引號到行尾都是註解，可跟程式碼同行 | `WRITE 'A'.  " 說明文字` |

```abap
* 整行註解：星號一定要貼在行首，縮排後的 * 會被當語法錯誤
WRITE 'Hello'.        " 行內註解：解釋這一行在做什麼
```

> 實務建議：註解寫「為什麼這樣做」，不要寫「這行在做什麼」（程式碼本身就看得出來的不用寫）。

## 4. 冒號鏈式寫法（Chained Statement)

多個 statement 開頭的關鍵字相同時，可以用冒號 `:` 合併，中間以逗號分隔、最後以句點收尾：

```abap
* 三句分開寫：
WRITE / 'A'.
WRITE / 'B'.
WRITE / 'C'.

* 鏈式寫法，跟上面完全等價：
WRITE: / 'A',
       / 'B',
       / 'C'.
```

展開規則：冒號前的部分會複製到每一段。所以：

```abap
DATA: gv_a TYPE i,
      gv_b TYPE i.
* 等價於
DATA gv_a TYPE i.
DATA gv_b TYPE i.
```

鏈式寫法在宣告區（`DATA:`、`TYPES:`）與輸出（`WRITE:`）最常用，是 ABAP 程式的「標準長相」，一定要會讀會寫。

## 5. 關鍵字不分大小寫

`write`、`Write`、`WRITE` 對編譯器完全相同（字面文字 `'...'` 裡的內容除外，那是資料，大小寫有差）。

```abap
write / 'lowercase 寫法'.
Write / 'Mixed Case 寫法'.
WRITE / 'UPPERCASE 寫法'.
```

團隊慣例靠 **Pretty Printer**（SE38 內按 `Shift+F1`）統一：可設定「關鍵字大寫、變數小寫」或全小寫等風格，寫完習慣按一下，全團隊格式一致。

## 6. WRITE 輸出

`WRITE` 把資料寫到「基本清單（basic list）」——執行畫面上看到的輸出。本階段所有練習都靠它觀察結果。

| 寫法 | 效果 |
|---|---|
| `WRITE '文字'.` | 接在目前輸出位置後面（同一行，間隔一格） |
| `WRITE / '文字'.` | 先換行，再從新行開頭輸出 |
| `WRITE: / 'A', 'B'.` | A 換行輸出，B 接在 A 後面 |

```abap
WRITE 'Hello ABAP!'.          " 第一行（清單起點）
WRITE '接在後面'.              " 不加 / ：同一行繼續
WRITE / '這是第二行'.          " 加 / ：換行
```

`WRITE` 後面可以接字面文字（`'...'`）、變數、系統欄位（如 `sy-datum` 今天日期），之後每一講都會用到；進階的位置與格式控制（欄位對齊、寬度）留到講義 12 列印排版。

## 7. 常見錯誤與陷阱

| 症狀 | 原因 |
|---|---|
| 錯誤訊息指在某行，那行看起來沒問題 | 上一行忘了句點 |
| `*` 註解報語法錯誤 | 星號不在行首（前面有空白） |
| 改了程式，執行結果沒變 | 只存檔沒啟用（Activate） |
| 鏈式寫法報錯 | 中間用了句點、或最後用了逗號 |
| 字面文字裡想放單引號 | 連寫兩個：`'It''s ok'` |
