# 釋放傳輸請求前檢查

依序執行，不可跳過：

1. 列出這個 Transport Request 底下包含的所有物件，跟使用者確認範圍正確（沒有夾帶不相關的物件）。
2. 對每個物件執行語法檢查（syntax check），回報是否全部通過。
3. 執行 ATC 檢查，列出所有 Error / Priority 1 項目；若有未排除的嚴重項目，**停止並回報，不要繼續**。
4. 確認是否有對應的測試類別，並回報是否為新增/未涵蓋的邏輯。
5. 搜尋是否有殘留的 `BREAK-POINT` 或除錯用 `WRITE` 輸出。
6. 以上都確認沒問題後，向使用者總結檢查結果，**明確詢問是否要釋放**。
7. 只有在使用者明確回覆「釋放」或「release」後，才呼叫釋放傳輸請求的動作。

若過程中任何一步驟有疑慮，優先回報給使用者而不是自行判斷跳過。

> 目前 `sap-adt` MCP 未暴露釋放/刪除傳輸請求的專屬工具（見 `.claude/rules/sap-adt-mcp.md` 第 9 節），這類動作只能透過該檔記錄的 ADT API workaround 手動執行，且一律要先給使用者確認。
